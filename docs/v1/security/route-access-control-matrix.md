## Phase 11 Notes

- Static review confirms report generation is admin-action only and gated by `payment_received`.
- Phase 11 hardens two reference-only paths: the legacy result page and report-request API.
- Phase 11 hardens admin page rendering so protected service-role reads occur only after `requireAdmin` succeeds.
- The patch intentionally keeps Next.js on 14.x and preserves the Next 14 `experimental.outputFileTracingIncludes` config shape.
- Live negative checks for unpaid report generation, logged-out report download, direct storage access and authenticated signed download still require a running deployment or local app with approved environment variables.
